SaaS vs On-Premises Deployment Plan

Version 1.0 2026-04-01 Planning

1. Overview

☁️ SaaS

  • We host everything
  • We own the API keys
  • Continuous upgrades
  • Our cloud (Supabase/AWS)
  • Subscription + usage pricing
  • We enforce metering (hard truth)

🏠 On-Premises

  • Customer hosts
  • Customer provides API keys
  • Customer-controlled upgrades
  • Customer's environment
  • Annual license fee
  • License agent or self-reported

2.3 Tenant Isolation (Row-Level Security)

-- RLS on newsletters table
ALTER TABLE newsletters ENABLE ROW LEVEL SECURITY;

CREATE POLICY "user_owns_newsletters"
  ON newsletters FOR ALL
  USING (auth.uid()::text = user_id);

-- Team plan: org members see all org newsletters
CREATE POLICY "org_members_see_newsletters"
  ON newsletters FOR SELECT
  USING (
    org_id IS NOT NULL AND
    org_id IN (
      SELECT org_id FROM org_members WHERE user_id = auth.uid()::text
    )
  );

2.4 SaaS Pricing Strategy

Revenue = MRR (subscriptions) + Overage (usage above plan limit)

MRR drivers:
  Free → Pro      (newsletter limit, email sending)
  Pro → Team      (seat limit, custom prompts feature)
  Team → Enterprise (SLA, data residency, SSO)

Overage drivers:
  Team plan: LLM token overage at $0.015/1K tokens
  Enterprise: negotiated rate
