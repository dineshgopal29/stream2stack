Licensing Plan

Version 1.1 2026-04-03 Partially Implemented

1. Overview

Stream2Stack uses two licensing strategies: SaaS (ToS + Stripe subscription, no key required) and On-Premises (cryptographic license key controlling features, seat limits, and expiry).

2. License Key Design

2.1 Key Structure (JWT-based)

Format: S2S-<base64url(header)>.<base64url(payload)>.<base64url(signature)>

Payload:
{
  "license_id":     "lic_abc123",
  "customer_id":    "cust_xyz789",
  "customer_name":  "Acme Corp",
  "plan":           "professional",
  "seats":          10,
  "issued_at":      1743724800,
  "expires_at":     1775260800,
  "features": {
    "custom_prompts": true,
    "audit_logs":     true,
    "sso":            false,
    "white_label":    false,
    "air_gap":        false
  }
}

2.2 License Validation

# backend/services/license.py
import jwt, os
from functools import lru_cache

_PUBLIC_KEY = os.getenv("S2S_LICENSE_PUBLIC_KEY", "")  # RSA-2048 PEM

@lru_cache(maxsize=1)
def validate_license() -> dict:
    raw_key = os.getenv("S2S_LICENSE_KEY", "")
    if not raw_key:
        raise LicenseError("S2S_LICENSE_KEY not set.")

    token = raw_key.removeprefix("S2S-")

    try:
        return jwt.decode(token, _PUBLIC_KEY, algorithms=["RS256"])
    except jwt.ExpiredSignatureError:
        raise LicenseError("License has expired. Renew at stream2stack.com/renew")
    except jwt.InvalidTokenError as e:
        raise LicenseError(f"Invalid license key: {e}")


def is_feature_enabled(feature: str) -> bool:
    try:
        return validate_license().get("features", {}).get(feature, False)
    except LicenseError:
        return False


def get_seat_limit() -> int | None:
    try:
        return validate_license().get("seats")
    except LicenseError:
        return 1  # fail closed

2.3 Hard Stop on Invalid License

# backend/main.py — lifespan startup
if os.getenv("S2S_DEPLOY_MODE") == "onprem":
    try:
        payload = validate_license()
        logger.info("License valid: plan=%s, expires=%s",
                    payload["plan"],
                    datetime.fromtimestamp(payload["expires_at"]).date())
    except LicenseError as e:
        logger.critical("LICENSE INVALID — server will not start: %s", e)
        raise SystemExit(1)

3. License Enforcement Points

3.1 Feature Gates

# Gate on a premium feature
@router.put("/prompts/system")
async def update_system_prompt(body: PromptUpdateRequest):
    if os.getenv("S2S_DEPLOY_MODE") == "onprem":
        if not is_feature_enabled("custom_prompts"):
            raise HTTPException(
                status_code=403,
                detail={
                    "code": "license_feature_unavailable",
                    "feature": "custom_prompts",
                    "upgrade_url": "https://stream2stack.com/upgrade",
                }
            )

3.2 Seat Enforcement

# backend/services/seat_manager.py
async def check_seat_available(user_id: str) -> None:
    limit = get_seat_limit()
    if limit is None:
        return  # unlimited plan

    active_count = await count_active_sessions()
    if active_count >= limit and not await user_has_active_session(user_id):
        raise HTTPException(
            status_code=402,
            detail={
                "code": "seat_limit_reached",
                "seats_used": active_count,
                "seats_licensed": limit,
            }
        )

4. License Issuance Workflows

4.1 Self-Serve (Online Purchase)

Customer visits stream2stack.com/on-prem │ ▼ Selects plan + seats → Stripe Checkout │ ▼ Stripe webhook → POST /license/issue {customer_id, plan, seats, duration_days} │ ▼ License Server: 1. Generate JWT payload 2. Sign with RSA-2048 private key 3. Store in license_records table 4. Email "S2S-<token>" + install guide │ ▼ Customer: sets S2S_LICENSE_KEY in .env

4.2 Enterprise / Air-Gapped

Sales agrees on plan + seats + duration │ ▼ Internal CLI (ops team): $ s2s-license issue \ --customer "Acme Corp" \ --plan enterprise \ --seats unlimited \ --days 365 \ --features sso,white_label,air_gap \ --output acme-license.txt │ ▼ License file emailed to customer Backed up in CRM + license_records DB

5. License Database (Our Side)

CREATE TABLE license_records (
    license_id       TEXT PRIMARY KEY,
    customer_id      TEXT NOT NULL,
    customer_name    TEXT NOT NULL,
    customer_email   TEXT NOT NULL,
    plan             TEXT NOT NULL,
    seats            INTEGER,           -- NULL = unlimited
    features         JSONB NOT NULL,
    issued_at        TIMESTAMPTZ NOT NULL,
    expires_at       TIMESTAMPTZ NOT NULL,
    renewed_from     TEXT REFERENCES license_records(license_id),
    revoked_at       TIMESTAMPTZ,
    revocation_reason TEXT,
    stripe_payment_id TEXT,
    created_at       TIMESTAMPTZ DEFAULT now()
);

6. Security & Anti-Tamper

ThreatMitigation
Key sharing between customerscustomer_id in payload; seat limit; telemetry agent reports hostname
Clock rollback attackLicense server validates issued_at is not in future; monotonic counters
JWT forgingRSA-2048 public key embedded in binary; private key never leaves license server
Running without networkOffline certs valid 90 days; renewal requires online check or manual CSV

7. Community Edition (CE) Strategy

Community Edition

Free
  • Apache 2.0 open-source
  • Single user
  • Ollama only
  • No email delivery
  • No web crawling
  • Self-hosted only

Commercial (On-Prem)

Standard → Enterprise
  • Proprietary license
  • All API providers
  • Email delivery
  • Multi-seat
  • SSO (Enterprise)
  • White-label (Enterprise)

SaaS

Free → Enterprise
  • ToS + Stripe
  • No license key
  • Quota gates
  • Hosted by us
  • Continuous updates
  • Usage-based overage

Dual-license model: CE code is open-source (Apache 2.0). Commercial additions (multi-seat, SSO, white-label, advanced analytics) are proprietary. Clear feature matrix at stream2stack.com/pricing drives top-of-funnel from CE to paid.
